# 贡献指南

感谢你为 `academic-deai-zh` 做贡献。

## 仓库定位

这是一个面向**中文论文与中文学术文本**的去 AI 味 skill。  
它不是英文版的解释镜像，而是**以英文版架构为母版、面向中文场景本地化的双生版**。

英文能力母版仓库：

- [zh4men9/academic-deai](https://github.com/zh4men9/academic-deai)

## 维护原则

- 英文版负责能力架构与风险模型
- 中文版负责中文论文语料、中文章节习惯、中文套话信号与中文输出接口
- 中文版应与英文版保持**结构强同步**
- 但中文规则不能靠逐句直译完成，必须针对中文学术写作重新表述

## 欢迎的贡献

- 改进中文学术场景下的触发描述与边界定义
- 改进中文论文常见 AI 痕迹的诊断规则
- 改进中文摘要、引言、相关工作、结论、附信和回复信的本地化示例
- 改进 benchmark/demo 的中文覆盖面
- 提高中文版与英文版在能力结构上的一致性

## 请避免

- 把中文版重新改回“中文说明的英文论文 skill”
- 机械直译英文版规则而不做中文语料适配
- 在未更新 `CHANGELOG.md` 的情况下修改默认行为
- 为了中文表达流畅而放松已有的风险边界
- 把高风险学术改写伪装成低风险润色

## 同步流程

当英文版发生变化时，建议按以下顺序同步：

1. 识别变更属于哪一类：
   - 结构同步：workflow、guards、edit levels、report buckets
   - 语料本地化：诊断词表、章节指南、示例、benchmark
2. 更新中文版 `SKILL.md`
3. 更新对应的 `references/`
4. 更新 `README.md`
5. 更新 `examples/` 与 `benchmark-demo/`
6. 更新 `CHANGELOG.md`
7. 运行校验脚本

## 最低验证要求

至少确认：

- skill 结构仍然合法
- 默认输出块全部为中文
- 风险报告仍明确区分：
  - `人工复核项`
  - `跳过的高风险项`
  - `未修改但可疑项`
- 仓库中不再出现“镜像版”“处理英文论文”等旧定位

## 发布说明

如果本次变更会影响用户对 skill 的理解、默认输出或风险边界，请在 `CHANGELOG.md` 中补一条记录。
